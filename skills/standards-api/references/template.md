## 2. URL Structure

```
/api/v1/{resource}/{id}/{sub-resource}
```

- Plural nouns: `/users`, `/orders`.
- Kebab-case multi-word: `/user-profiles`.
- IDs are opaque strings (UUID or ULID).

## 3. Methods and Status Codes

| Method | Use | Success | Failures |
|--------|-----|---------|----------|
| GET | List or fetch | 200 | 400, 401, 403, 404 |
| POST | Create / action | 201 (with `Location`) | 400, 401, 403, 409, 422 |
| PUT | Replace | 200 | 400, 401, 403, 404, 409 |
| PATCH | Partial update | 200 | 400, 401, 403, 404, 409 |
| DELETE | Remove | 204 | 401, 403, 404 |

Validation failures: **400** (syntax) or **422** (semantic). Pick one and stick to it: this project uses {{400|422}}.

## 4. Request Envelope

Required headers:
- `Content-Type: application/json`
- `Authorization: Bearer <token>` for protected routes
- `Idempotency-Key: <uuid>` for non-GET methods that create state
- `X-Request-ID` (correlation, generated by client or gateway)

## 5. Response Envelope

### Success

```json
{
  "data": { ... },
  "meta": { "page": 1, "perPage": 25, "total": 1234 }
}
```

### Error

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input",
    "details": [
      { "field": "email", "message": "Invalid format" }
    ]
  }
}
```

Field naming: **{{camelCase | snake_case}}** — same on request and response.

## 6. Stable Error Codes

| Code | HTTP | Meaning |
|------|------|---------|
| `VALIDATION_ERROR` | 422 | Input failed validation |
| `NOT_FOUND` | 404 | Resource missing |
| `UNAUTHORIZED` | 401 | No / invalid auth |
| `FORBIDDEN` | 403 | Auth ok, not allowed |
| `CONFLICT` | 409 | Concurrent edit / unique violation |
| `RATE_LIMITED` | 429 | Quota exceeded |
| `INTERNAL_ERROR` | 500 | Server fault |

## 7. Pagination

Cursor-based:

```
GET /users?cursor=<opaque>&limit=25
```

Response includes `meta.nextCursor`; absent means no more pages.

## 8. Versioning

- URL prefix: `/v1`, `/v2`, etc.
- Breaking changes → new major version + deprecation header on prior:
  ```
  Deprecation: true
  Sunset: <RFC1123 date>
  ```
- Additive (new optional field, new endpoint) → ship on the current version.
- Removal: minimum {{N}} months between deprecation and removal.

## 11. Idempotency

POST endpoints that create state accept `Idempotency-Key`. Replay window: {{24h}}. Same key + same body → same response. Same key + different body → 422.
