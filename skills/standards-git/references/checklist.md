# Git Standards — Validation Checklist

- [ ] Branching model named, with the reason.
- [ ] Branch-naming patterns for every branch type, with two examples each.
- [ ] Commit-message format with at least one example per type.
- [ ] PR / MR process: required approvals, review responsibilities, blocking checks.
- [ ] Merge strategy per branch type.
- [ ] Branch protection on `main` enumerated.
- [ ] Tagging convention and changelog rule.
- [ ] Hotfix flow with allowed trigger, approver, and compensating check.
- [ ] File size 8-15 KB.
- [ ] No `{{placeholders}}` left.
